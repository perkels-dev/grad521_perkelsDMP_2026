# Data Management Plan
## Data Description
The research question this project is trying to answer is, generally, how animals sense stimuli. The field has identified proteins that are important for sensing the environment but how those proteins do that is sometimes still unknown. For this project, the lab is trying to solve the structure of some of these proteins in order to better understand their function.

The data that has been and will be collected are all experimental - some data has already been collected but much has not been collected yet. Most of the data are and will be spreadsheets, images, and videos. This project will also generate specialized data formats like .tar, .mmCIF, and .map for the micrograph and 3D model data - these data have not yet been collected.
The majority of the spreadsheet data has and will be collected with an HPLC (high performance liquid chromatography) instrument. Some of the spreadsheet data is generated by the mass spectrometry resource at OHSU. The video data has and will be collected with a video camera. The image data has been collected with an instrument that images gels (a LiCor gel imager, used to analyze protein). The structural data will be generated at a cryo-EM center (likely the Pacific Northwest Cryo-EM Center or PNCC). The majority of that data will be processed on their supercomputer and stored through the Pacific Northwest National Lab (PNNL) and the Protein Databank/EM Database (PDB/EMDB).

Types of data:
1.	Spreadsheet data – 1-2MB
2.	Image and video data – <1MB
3.	Structural data – 2-5TB

Analysis of the spreadsheet data and structural data will require coding scripts which will be both R and bash scripts. 
Physical samples will also be processed and generated in the course of this project.

## Roles and responsibilities
The research data must comply with Oregon State University's policies on data management as well as the National Institutes of Health, who is the funder of this project. This research is being conducted at OSU and uses animal tissue samples, so the datasets created over the course of this project are required by the university to be preserved for at least 3 years after project completion. Additionally, the NIH is the funder of this project and requires that all research data (i.e. data needed to validate and replicate experiments) needs to be both preserved for 3 years (in agreement with OSU policy) and shared as openly as possible. The primary individual responsible for keeping this project compliant with the above goals is the PI, who does this by guiding the graduate student during their time on the project. The graduate student is responsible for maintaining the data during their time on the project and keeping it organized and properly documented according to the PI's guidance. This allows the PI then takes over preservation of the data once the graduate student leaves.

•	archiving and preservation – the PI, primarily, the graduate student also play a role here to prepare data for preservation

•	data manager – the graduate student, but will pass to the PI when the graduate student leaves

•	instrumentation maintenance – primarily the PI, graduate students also perform routine upkeep and maintanance

•	data collection/data generation – the graduate student 

•	data organization – the graduate student, will pass to the PI after the graduate student leaves

•	metadata generation – the PI and graduate students

•	quality control – the PI and graduate students, primarily the graduate students will perform QC at time of data collection but the PI will also assess quality throughout the data lifecycle

•	data analysis – the PI and graduate students, primarily the graduate students but the PI will guide analysis

•	DMP implementation – the PI will primarily guide DMP implementation but the graduate students will be responsible for the day-to-day work of implementation

•	access control – the PI and graduate students

•	software creation and maintenance – the graduate student

Management of the physical samples will be handled by the graduate student at OSU. Grids that get sent for analysis are kept by PNCC for up to 6 months. Purified protein sent out for analysis at the proteomics shared resource at OHSU and is managed by that facility - they largely don't keep samples and instead analyze them as soon as possible.

## Data standards and metadata

For the final structural data (i.e. the final models generated from this work), the lab will use the PDB metadata standard. Files output from the processing of raw data will be in the PDB/mmCIF format/standard. The raw micrograph data will be deposited with a metadata.txt file automatically generated by the PNCC that contains data collection parameters. For the rest of the data, there aren't really structured metadata standards that are applicable to these data, so the graduate student will keep these files organized with descriptive naming conventions (for example: yyyymmdd-expID) and accompany them with the following kinds of unstructured metadata. For the spreadsheet data, the graduate student will record the instrument used to collect the data, the software used to process it, and a description of the variable names in a readme.txt file. For the image and video data, the graduate student will keep a log file with the same file name (i.e. 20260511_bindingtime_wb.log to accompany 20260511_bindingtime_wb.tif) that describes what the image or video is, what experiment it is associated with, and basic parameters (i.e. what instrument was used to collect the image data). 

The data shouldn’t really need version control, but the lab plans to use careful naming conventions to keep track of versions.

For the code that will be generated for this project, the graduate student will annotate throughout the raw code to describe what the code was designed for and what each block is supposed to do.

For the physical samples, the graduate student will maintain a log file that describes the what the sample is (i.e. purified protein, tissue, etc), where it is stored, and the day it was last accessed (all physical samples are frozen and the number of times a sample is thawed/refrozen is important for sample integrity). This log will likely be a spreadsheet with its own descriptive readme.txt to describe it.

## Storage and security

None of the datasets that have been or will be generated by this project need to be protected so to share them, the lab will make them publicallly available as follows:

•	when it will be available – for the model and raw micrograph data, up to a year or so prior to publication; for the image, video, and graphing data, upon publication 

•	how and where will the data be made available – for the model data, either in the PDB or EMDB (which are repositories for structural data, there is a possibility that the data will end up in both); for the raw micrographs needed to generate the model data, these will be stored on the MyEMSL archive; for the image, video, and graphing data, the lab will deposit it in an open repository like Zenodo.


Data will be preserved by the lab for at least 3 years after the completion of this project, as required by both OSU and the NIH. The data will be stored on several lab computers locally and on the cloud (via OneDrive to allow access to the lab). For the very large datasets generated for structural analysis, model datasets will be stored by the PDB repository and the raw micrograph datasets will be stored by the MyEMSL archive at PNNL. 

The lab plans to further backup the spreadsheet, image, and video data (that is stored locally) to external hard-drives which will stay in the lab space. For the very large datasets, those are managed and backed up by the PDB and MyEMSL repositories.

Physical samples will be stored in the general lab freezer. Some of these samples are at biosafety level 2 (BSL-2) but are not required to be especially secure, so they will only be kept physically separate from other samples. Final purified protein will be also stored in the general lab freezer until needed, while final grids will be kept in a specialized dewar until they are shipped to the PNCC for screening and data collection.

## Access and data sharing

Model data will be made available in the common PDB/mmCIF format (for the model) and EM Map format (for the processed volume data, this is not quite the raw data files but it is commonly used to validate published models for cryo-EM). Raw micrograph data will be available as .tar files, which is standard for these data. Image data will be deposited as .tif files (which is the highest quality commonly available), video data will be deposited in image stacks of .tif files, and the graph data will be deposited as both raw and processed data – for the image, video, and graph data, there will be an accompanying readme file with metadata.
The model data will be under a CC0 license, so it will be as openly available as possible, as is standard for the PDB. The image, video, and graphing data will likely be deposited with a CC-BY license, so it will be openly available but use would need to be attributed to us.

Physical samples will only be accessed by those in this lab, primarily the PI or the graduate studet. Final purified samples will be sent outside OSU for other analyses and will be only accessed by personnel at those facilities.

## Archiving and preservation

The PDB/EMDB has special protections, and some version of it will likely remain preserved into perpetuity due to the interest and funding from worldwide governments. The MyEMSL archive has plans to maintain data into perpetuity as tape archives but the digital data will be phased out after access has reached 90% capacity. The remaining data will be preserved by the repository selected; for example, Zenodo is estimated to be operational for at least the next 20 years. Access to the data stored in repositories will be maintained by the repositories themselves.
The model data will be preserved as .pdb or .cif formats, the EM map data will be preserved as .map files, the micrograph data will be .tar files, the images and videos will be .tif files, and the graphing data will be both .cdf (an unusual file format) and .csv (this format is much more amenable for preservation) files.

Physical samples not used in this project will be maintained in long-term storage in this lab until another project needs those materials.
